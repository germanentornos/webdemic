Why validate your HTML code?

As a beginner it is very common that you will make mistakes in writing your HTML code. Incorrect or non-standard code may cause unexpected results in how your page is displayed or functions in browsers.

To prevent this you can test or validate your HTML code against the formal guidelines and standards set by the Wide Web Consortium (W3C) for HTML/XHTML web pages. The World Wide Web Consortium provides a simple online tool (https://validator.w3.org/) that automatically checks your HTML code and points out any problems or errors your code might have, such as missing closing tags or missing quotes around attributes.

Data validation is intended to provide certain well-defined guarantees for fitness, accuracy, and consistency for any of various kinds of user input into an application or automated system.

Some of the reasons for validating a website include:

  • Helping to create web pages that are cross-browser, cross-platform compatible. These websites are also likely to be compatible with future versions of web browsers and standards.
  • Increasing visibility in search engines.
  • Reducing unexpected errors.
  • Making web pages more accessible to visitors.
